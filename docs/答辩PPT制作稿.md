# DJ 音乐电台答辩 PPT 制作稿

本文按老师给的《答辩PPT示例.pptx》扩展为可直接制作 PPT 的逐页稿。示例 PPT 只有“系统演示、截图、分工、组员工作介绍”等骨架；这里按当前真实项目补全页数、页面内容、讲解重点和建议截图。

## 制作原则

- 不要把 PPT 做成纯文字说明书，每页保留 3-5 个重点；
- 技术页用架构图、流程图、表格表达；
- 演示页用真实系统截图，至少 6 张；
- 不夸大未实现能力：验证码、找回密码、真实头像上传、Gateway 全局鉴权还没有完成；
- 队员 A 的分工必须写明“模块 1 + 整合统筹 + merge + 全局 bug 修复 + 文档协调”。

## 第 1 页：封面

标题：DJ 音乐电台项目答辩

副标题：基于 Spring Cloud + Vue3 的智能音乐电台与推荐系统

页面内容：

```text
项目名称：DJ 音乐电台
团队成员：队员 A、队员 B、队员 C、队员 D
指导老师：按实际填写
答辩时间：按实际填写
```

设计建议：背景使用系统首页截图的弱化版或深色控制台风格背景，标题居中。

## 第 2 页：项目背景与目标

标题：为什么做 DJ 音乐电台

页面文字：

```text
问题 1：普通播放器只能搜索和播放，无法用自然语言表达复杂场景需求。
问题 2：普通 AI 能聊天，但直接推荐歌曲容易编造，结果不一定能播放。
问题 3：推荐、播放、歌单、天气、用户行为通常分散在不同页面。

目标：把 AI 对话、真实歌曲数据、推荐算法、天气时间和用户行为整合到一个单屏音乐控制台中。
```

讲解重点：项目不是简单播放器，而是“AI 助手 + 音乐系统工具”的结合。

## 第 3 页：系统整体效果

标题：单屏 DJ 控制台

放图：系统首页全屏截图。

标注：

```text
顶部：时间、天气、定位
左侧：AI DJ 对话
中间：播放器、歌曲、歌单、歌词、情绪
右侧：热门榜、今日推荐、偏好标签
底部：用户登录态、收藏、历史、账号操作
```

讲解重点：所有核心功能都在第一屏完成，适合答辩现场演示。

## 第 4 页：技术架构

标题：Spring Cloud 微服务架构

页面图示：

```text
Vue3 / Vite :5173
  -> Gateway :8080
    -> module-chat :8081
    -> module-music :8082
    -> module-rec :8083
    -> module-user :8084

基础设施：
Nacos、MySQL、Redis、RabbitMQ、NeteaseCloudMusicApi、DeepSeek、QWeather
```

页面文字：

```text
Gateway 负责统一路由
Nacos 负责服务注册发现
MySQL 保存业务数据
Redis 保存热门榜
RabbitMQ 支撑异步导入和推荐通知
DeepSeek 和 QWeather 提供外部智能与天气能力
```

## 第 5 页：团队分工

标题：团队分工与协作

页面表格：

| 成员 | 模块 | 工作内容 |
|:--|:--|:--|
| 队员 A | 智能对话 DJ + 整合统筹 | chat、天气、时间、定位、DeepSeek 工具、前端 Chat；合并 music/rec/user，整体 bug 修复，一键脚本和文档 |
| 队员 B | 音乐中心 | music、网易云代理、播放器、歌单、导入、歌词、情绪、收藏、历史、播放队列 |
| 队员 C | 推荐系统 | rec、行为上报、Redis 热榜、每日推荐、相似推荐、偏好标签、推荐刷新 |
| 队员 D | 用户中心 | user、注册登录、JWT、资料、修改密码、收藏/历史入口 |

讲解重点：A 不只是 chat 模块，还负责最终整合和质量兜底。

## 第 6 页：模块 1 智能对话 DJ

标题：AI 对话不是模板回复

页面流程：

```text
用户输入
  -> DeepSeek 规划工具
  -> 后端执行 music / rec / weather / time / location
  -> DeepSeek 基于真实结果组织回复
  -> 返回可点击歌曲
```

页面文字：

```text
支持普通聊天、音乐推荐、天气查询、时间查询、当前位置工具。
AI 不能随意编造歌曲，推荐歌曲必须来自系统工具结果。
WebSocket 支持实时对话和浏览器定位工具请求。
```

建议截图：问“我这里天气怎么样，适合听什么”，显示定位/天气/推荐歌曲。

## 第 7 页：模块 2 音乐中心

标题：音乐播放与网易云联动

页面文字：

```text
本地歌库：歌曲列表、搜索、详情、流派筛选
网易云代理：搜索、播放 URL、歌词、歌单、封面
歌单管理：创建、编辑、删除、排序、导入
播放体验：队列、播放模式、刷新后恢复、封面播放详情
用户数据：收藏、历史、用户品味
```

建议截图：网易云搜索结果、播放器和队列展开。

## 第 8 页：歌词情绪分析

标题：歌词情绪与用户品味

页面流程：

```text
歌词 LRC
  -> DeepSeek 情绪分析
  -> 失败时关键词规则兜底
  -> song_emotion
  -> song.emotion_tags
  -> user_taste
```

页面文字：

```text
情绪画像包含主情绪、次情绪、效价、唤醒度、情绪强度、适合场景。
推荐模块会优先使用 emotion_tags，避免 genre 只有“网易云”时推荐过粗。
```

建议截图：歌曲情绪信息或歌单情绪总览。

## 第 9 页：模块 3 推荐系统

标题：从用户行为到每日推荐

页面流程：

```text
播放/收藏/分享
  -> /rec/behavior
  -> MySQL user_behavior
  -> Redis song:hot:rank
  -> 热门榜

刷新今日推荐
  -> 情绪标签
  -> 有效流派
  -> 歌手/专辑/时长
  -> 协同过滤
  -> daily_recommend
```

页面文字：

```text
play +1，like +3，share +2，skip +0。
支持手动刷新今日推荐，便于答辩现场演示。
Feign 调 music 补全真实歌名和歌手，避免只显示“歌曲#ID”。
```

建议截图：右侧推荐面板和刷新按钮。

## 第 10 页：模块 4 用户中心

标题：用户会话与跨模块数据归属

页面文字：

```text
已完成：注册、登录、JWT、用户资料、修改密码。
收藏和历史不在 user 模块重复存储，而是统一复用 music 模块数据。
登录态变化后，Chat、Music、Rec 会按当前用户 ID 刷新。
```

```text
UserBar 登录
  -> 保存 JWT 和用户资料
  -> dj-user-session-changed
  -> ChatPanel / MusicPanel / RecPanel 刷新
```

建议截图：登录弹窗、用户菜单、修改密码弹窗或收藏历史弹窗。

## 第 11 页：数据库设计

标题：核心数据表

页面表格：

| 数据库 | 表 | 用途 |
|:--|:--|:--|
| mall | chat_history | 对话历史 |
| mall | dj_user | 用户身份 |
| mall | user_behavior / daily_recommend | 行为和推荐 |
| dj_music_radio | song / playlist / playlist_song | 歌曲和歌单 |
| dj_music_radio | user_favorite / play_history | 收藏和历史 |
| dj_music_radio | user_play_queue / user_player_state | 播放队列和播放模式 |
| dj_music_radio | song_emotion / emotion_keyword / user_taste | 情绪和品味 |

讲解重点：为什么收藏、历史、队列由 music 统一维护，避免 user 和 music 数据冲突。

## 第 12 页：系统演示路线

标题：现场演示流程

页面步骤：

```text
1. 一键启动项目，打开 http://127.0.0.1:5173/
2. 展示首页、时间、天气悬浮详情
3. 登录或注册用户
4. 与 AI 对话，触发天气/音乐工具
5. 点击聊天推荐歌曲播放
6. 网易云搜索或导入歌单
7. 展示歌词和情绪分析
8. 播放/收藏后刷新推荐
9. 查看收藏、历史和播放队列恢复
```

讲解重点：控制时间在 2 分钟左右，优先演示最能体现联动的链路。

## 第 13 页：项目亮点

标题：项目亮点

页面文字：

```text
1. AI 工具化：DeepSeek 负责理解和回复，事实数据来自系统工具。
2. 微服务分工：chat、music、rec、user、gateway 边界清晰。
3. 真实外部服务：网易云代理、和风天气、DeepSeek。
4. 推荐闭环：播放/收藏行为影响热榜、偏好和每日推荐。
5. 用户联动：登录态驱动聊天、音乐、推荐、收藏、历史和播放队列。
6. 答辩友好：一键启动、关闭脚本和完整运行文档。
```

## 第 14 页：问题与改进

标题：当前限制与后续优化

页面文字：

```text
当前限制：
- Gateway 还没有全局鉴权过滤器。
- 用户模块没有验证码、找回密码和真实头像上传。
- 网易云播放 URL 受版权、VIP、地区限制影响。
- 推荐效果依赖足够的歌曲数据和用户行为。

后续优化：
- 补 Gateway 鉴权和 token 刷新。
- 接邮箱验证码、找回密码、头像上传。
- 扩充歌曲和情绪数据。
- 统一前端 API 代理，减少开发直连。
```

标题：总结

页面文字：

```text
DJ 音乐电台实现了一个可运行、可演示、可扩展的智能音乐系统。
项目完成了 AI 对话、音乐播放、网易云代理、推荐算法、用户中心和前端单屏控制台的整合。
相比普通播放器，本项目重点体现了微服务架构、AI 工具调用、用户行为推荐和跨模块联动。
```
