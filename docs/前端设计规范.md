# 前端设计规范

本文档按当前 `frontend/src/App.vue` 单屏控制台整理。项目真实入口不是路由页面，而是 App 直接组合各业务组件。

## 1. 页面结构

```text
App.vue
├── 顶栏
│   ├── 项目标题
│   ├── TimeWidget.vue
│   └── WeatherWidget.vue
├── 主区域
│   ├── ChatPanel.vue
│   ├── MusicPanel.vue
│   └── RecPanel.vue
└── 底栏
    └── UserBar.vue
```

布局目标：桌面 DJ 控制台，一屏完成主要操作。不要把主入口改成营销页或传统多页面后台。

## 2. 组件边界

| 组件 | 当前职责 |
|:--|:--|
| `ChatPanel.vue` | WebSocket/REST 对话、工具请求、定位上下文、推荐歌曲点击播放 |
| `WeatherWidget.vue` | 顶栏天气、浏览器定位、实时/演示数据标识、悬浮详情 |
| `TimeWidget.vue` | 顶栏时间日期、悬浮完整时间信息 |
| `MusicPanel.vue` | 播放器容器、歌曲列表、歌单、歌词、情绪、外部歌曲播放事件 |
| `PlayerCore.vue` | 播放控制、进度、音量、当前歌曲、封面点击完整播放详情 |
| `SongList.vue` | 本地歌库和网易云搜索结果 |
| `PlaylistPanel.vue` | 歌单列表、歌单歌曲、导入入口 |
| `LyricsPanel.vue` | 歌词展示 |
| `EmotionInfo.vue` / `EmotionTag.vue` | 歌曲情绪和品味信息 |
| `RecPanel.vue` | 热门榜、今日推荐、偏好标签 |
| `UserBar.vue` | 底部用户状态、真实登录注册、退出、收藏/历史弹窗 |

`frontend/src/views/*` 和 `router/index.js` 当前多为旧占位，不是主演示入口。

## 3. API 调用规范

统一从 `frontend/src/api/` 访问后端：

| 文件 | 用途 |
|:--|:--|
| `api/index.js` | Axios 实例，默认走 `/api` |
| `api/chat.js` | chat 接口 |
| `api/music.js` | music 接口，部分路径直连 `/music` |
| `api/rec.js` | rec 接口 |
| `api/user.js` | user 登录注册、JWT token、本地会话、收藏/历史入口 |

Vite 代理：

```text
/api/**   -> http://localhost:8080/**
/music/** -> http://localhost:8082/music/**
```

WebSocket：

```text
ws://当前页面host:8080/chat/ws
```

## 4. 跨组件事件

聊天推荐歌曲点击播放使用浏览器事件：

```js
window.dispatchEvent(new CustomEvent('dj-play-song', { detail: song }))
```

`MusicPanel.vue` 监听 `dj-play-song`，把 `selectedSongs` 转成播放器需要的结构。网易云歌曲会继续获取播放 URL 和歌词，本地歌曲按已有 `filePath` 或后端详情播放。

播放队列状态由 `MusicPanel.vue` 保存到 `/music/queue/state`。刷新页面、切换登录用户后，应按当前用户恢复 `currentSong`、`playQueue` 和 `playMode`。

用户会话变化使用浏览器事件：

```js
window.dispatchEvent(new CustomEvent('dj-user-session-changed', { detail: user }))
```

`ChatPanel.vue`、`MusicPanel.vue`、`RecPanel.vue` 监听该事件，切换用户后刷新聊天历史、收藏状态、播放历史、推荐和偏好。

## 5. 样式约定

全局样式位于：

```text
frontend/src/styles/variables.css
frontend/src/styles/global.css
```

优先使用 CSS 变量：

```css
var(--color-bg)
var(--color-surface)
var(--color-surface-hover)
var(--color-border)
var(--color-primary)
var(--color-accent)
var(--color-success)
var(--color-info)
var(--color-text)
var(--color-text-muted)
```

组件内可以写局部样式，但新增颜色、阴影、圆角、间距时要尽量与现有控制台风格一致。

## 6. 交互规范

- 图标按钮必须有清晰的 hover/focus 状态；
- 工具类信息适合用悬浮详情，例如天气详情、完整时间；
- 聊天中的推荐歌曲可以点击，点击后应进入播放器；
- Chat 面板放大/收回、播放详情打开/关闭应保留过渡动画，避免界面闪变；
- 后端真实数据不可用时，前端要显示明确状态，不要伪装成成功；
- 列表为空、加载中、接口失败都要有可理解的 UI 状态；
- PC 优先，小屏可以降为纵向布局，但不要求做完整移动 App。

## 7. 用户模块注意事项

`UserBar.vue` 当前已经调用真实 user 接口。登录成功后：

- token 存到 `localStorage` 的 `dj-user-token`；
- 用户资料存到 `dj-user-profile`；
- `api/music.js`、`api/rec.js` 默认读取当前用户 ID；
- 收藏和播放历史入口走 `module-user`，实际数据仍由 `module-music` 统一维护。
