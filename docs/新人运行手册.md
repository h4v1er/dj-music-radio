# 新人运行手册

本文档按当前仓库真实代码编写，目标是让新队员从零拉仓库后能知道要装什么、配什么、启动什么、怎么判断问题。

## 1. 克隆仓库

```bash
git clone git@github.com:h4v1er/dj-music-radio.git
cd dj-music-radio
git checkout dev-chat
```

团队当前集成分支是 `dev-chat`，已经整合 chat、music、rec、user 的阶段性实现。

## 2. 必需环境

| 环境 | 当前远端版本 | 用途 |
|:--|:--|:--|
| JDK | 25 用于当前 POM 构建，17 仍是远端默认 | Spring Boot 多模块构建 |
| Maven | 3.9.16 | 后端构建 |
| Node.js / npm | 22.15.1 / 10.9.2 | 前端、网易云 API 代理 |
| MySQL | MySQL80 | 业务数据 |
| Redis | 8.8.0 | 推荐热榜 |
| RabbitMQ | 4.3.1 | 歌单导入、推荐通知 |
| Nacos | 2.3.2 standalone | 服务注册发现 |

当前远端 Windows 已准备这些环境。普通本机运行时，需要自行安装同等环境。

## 3. 环境变量

至少配置 MySQL 密码：

```powershell
[Environment]::SetEnvironmentVariable("MYSQL_PASSWORD", "你的MySQL root密码", "User")
```

如果要启用真实 AI、天气和自定义 JWT 密钥，再配置：

```powershell
[Environment]::SetEnvironmentVariable("DEEPSEEK_API_KEY", "你的DeepSeek Key", "User")
[Environment]::SetEnvironmentVariable("DEEPSEEK_API_URL", "https://api.deepseek.com/v1/chat/completions", "User")
[Environment]::SetEnvironmentVariable("DEEPSEEK_MODEL", "deepseek-chat", "User")

[Environment]::SetEnvironmentVariable("QWEATHER_API_KEY", "你的和风天气Key", "User")
[Environment]::SetEnvironmentVariable("QWEATHER_API_HOST", "你的和风天气专属Host", "User")

[Environment]::SetEnvironmentVariable("JWT_SECRET", "至少32字节的用户JWT密钥", "User")
[Environment]::SetEnvironmentVariable("JWT_EXPIRE_MS", "86400000", "User")
```

RabbitMQ 默认使用 `guest/guest`，如本机不同：

```powershell
[Environment]::SetEnvironmentVariable("RABBITMQ_USERNAME", "guest", "User")
[Environment]::SetEnvironmentVariable("RABBITMQ_PASSWORD", "guest", "User")
```

改完用户级环境变量后，要重启终端或重启服务进程。

## 4. 数据库初始化

当前代码使用两个数据库：

| 数据库 | 使用模块 | SQL |
|:--|:--|:--|
| `mall` | `module-chat`、`module-rec`、`module-user`、旧模板代码 | `module-chat/src/main/resources/sql/chat_history.sql`、`module-rec/src/main/resources/sql/rec_tables.sql`、`module-user/src/main/resources/sql/user_tables.sql` |
| `dj_music_radio` | `module-music` | `module-music/src/main/resources/init.sql`、`emotion_schema.sql`、`emotion_keywords.sql` |

建议执行：

```sql
CREATE DATABASE IF NOT EXISTS mall DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
CREATE DATABASE IF NOT EXISTS dj_music_radio DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
```

然后执行：

```text
module-chat/src/main/resources/sql/chat_history.sql
module-rec/src/main/resources/sql/rec_tables.sql
module-user/src/main/resources/sql/user_tables.sql
module-music/src/main/resources/init.sql
module-music/src/main/resources/emotion_schema.sql
module-music/src/main/resources/emotion_keywords.sql
```

`module-rec/src/main/resources/sql/rec_tables.sql` 内容等价于：

```sql
CREATE TABLE IF NOT EXISTS user_behavior (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  song_id INT NOT NULL,
  action VARCHAR(20) NOT NULL,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_action (user_id, action),
  INDEX idx_song_action (song_id, action)
);

CREATE TABLE IF NOT EXISTS daily_recommend (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  song_id INT NOT NULL,
  reason VARCHAR(200),
  push_date DATE NOT NULL,
  INDEX idx_user_date (user_id, push_date)
);
```

## 5. 启动中间件

如果是在已经配置好的答辩机 / 远端 Windows 工作区，优先使用项目一键脚本，脚本会启动 MySQL、RabbitMQ、Redis、Nacos、网易云 API、Gateway、四个后端模块和前端：

```powershell
cd D:\projects\dj-music-radio
powershell -ExecutionPolicy Bypass -File .\scripts\windows\start-dj-music-radio.ps1
```

关闭整套环境：

```powershell
cd D:\projects\dj-music-radio
powershell -ExecutionPolicy Bypass -File .\scripts\windows\stop-dj-music-radio.ps1
```

保留中间件，只关闭项目服务：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\windows\stop-dj-music-radio.ps1 -KeepMiddleware
```

也可以双击运行：

```text
scripts\windows\start-dj-music-radio.cmd
scripts\windows\stop-dj-music-radio.cmd
```

下面是手动分步启动方式。

远端 Windows 已有脚本：

```powershell
powershell -ExecutionPolicy Bypass -File D:\devtools\start-dev-env.ps1
powershell -ExecutionPolicy Bypass -File D:\devtools\check-dev-env.ps1
```

确认：

```text
MySQL:     127.0.0.1:3306
Redis:    127.0.0.1:6379
RabbitMQ: 127.0.0.1:5672
Nacos:    127.0.0.1:8848
```

RabbitMQ 管理页面：`http://127.0.0.1:15672/`，默认 `guest/guest`。

## 6. 安装前端依赖

```powershell
cd D:\projects\dj-music-radio\frontend
npm.cmd install
```

## 7. 启动网易云 API 代理

`module-music` 的网易云搜索、播放 URL、歌词、歌单导入依赖独立 Node 服务 `NeteaseCloudMusicApi:3000`。

```powershell
cd D:\projects\dj-music-radio
powershell -ExecutionPolicy Bypass -File scripts\netease\start-netease-api.ps1
```

脚本会把依赖安装到 `.runtime/netease-api/`，该目录已被 `.gitignore` 忽略。

检查：

```powershell
Invoke-RestMethod "http://127.0.0.1:3000/search?keywords=test&limit=1"
```

## 8. 构建后端

当前父 POM 写的是：

```xml
<java.version>25</java.version>
```

远端默认 `java` 是 17，所以构建时使用 JDK25 脚本：

```powershell
cd D:\projects\dj-music-radio
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 clean package -DskipTests
```

如果本机只有 JDK17，要么安装 JDK25，要么全队统一把 POM 改回 17 后再构建。不要各自随意改版本后提交。

## 9. 启动后端

可以分别启动：

```powershell
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 spring-boot:run -pl gateway
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 spring-boot:run -pl module-chat
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 spring-boot:run -pl module-music
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 spring-boot:run -pl module-rec
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 spring-boot:run -pl module-user
```

远端也有打包后启动脚本和计划任务脚本：

```powershell
powershell -ExecutionPolicy Bypass -File D:\projects\setup-dj-service-tasks.ps1
powershell -ExecutionPolicy Bypass -File D:\projects\stop-dj-backend-services.ps1
```

```powershell

```

浏览器访问：


## 11. 验证接口

```powershell
Invoke-RestMethod "http://127.0.0.1:8080/chat/hello"
Invoke-RestMethod "http://127.0.0.1:8080/music/hello"
Invoke-RestMethod "http://127.0.0.1:8080/rec/hello"
Invoke-RestMethod "http://127.0.0.1:8080/user/hello"
Invoke-RestMethod "http://127.0.0.1:8080/chat/weather?city=北京"
Invoke-RestMethod "http://127.0.0.1:8080/music/netease/ping"
```

用户注册登录验证：

```powershell
$body = @{ username = "demo_user"; password = "test123456"; nickname = "Demo" } | ConvertTo-Json
Invoke-RestMethod -Method Post "http://127.0.0.1:8080/user/register" -ContentType "application/json" -Body $body
$login = Invoke-RestMethod -Method Post "http://127.0.0.1:8080/user/login" -ContentType "application/json" -Body (@{ username = "demo_user"; password = "test123456" } | ConvertTo-Json)
Invoke-RestMethod "http://127.0.0.1:8080/user/info" -Headers @{ Authorization = "Bearer $($login.data.token)" }
```

## 12. 常见问题

| 现象 | 原因 | 处理 |
|:--|:--|:--|
| Maven 报 JDK 版本不支持 | 默认 Java 是 17，但 POM 是 25 | 用 `mvn25-dj-music-radio.ps1` 或统一版本 |
| 前端能打开但接口失败 | 后端/Gateway 未启动或 Vite 代理目标不可达 | 先测 `/chat/hello` 等接口 |
| 网易云搜索失败 | 3000 代理没启动 | 启动 `scripts/netease/start-netease-api.ps1` |
| 聊天能回但不智能 | 未配置 `DEEPSEEK_API_KEY` 或调用失败 | 配环境变量并重启 `module-chat` |
| 天气显示演示数据 | 未配置和风天气 key/host 或接口失败 | 配 `QWEATHER_API_KEY` / `QWEATHER_API_HOST` |
| 本地歌库搜索为空 | `song` 表没数据或导入乱码 | 通过网易云歌单导入，检查 MySQL 编码 |
| 推荐榜为空 | 没有上报播放/收藏行为 | 调 `/rec/behavior` 或前端播放后再看 |
| 用户注册失败 | `mall.dj_user` 表没建或用户名重复 | 执行 `module-user/src/main/resources/sql/user_tables.sql`，换一个用户名重试 |
| 用户收藏/历史为空 | music 模块没有对应用户行为或 module-music 未启动 | 先启动 module-music，播放/收藏歌曲后再查看 |
