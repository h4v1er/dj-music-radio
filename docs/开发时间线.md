## Week 1：基础设施（目标：各自服务能启动）

### Day 1（周一）

| 队员 | 任务 | 产出 |
|:---|:-----|:-----|
| **D** | 🔥 **搭 Nacos + Gateway，全队注册上** | Nacos 服务列表能看见 5 个服务 |
| A | 建 `chat_history` 表，写实体类 | 表建好，Admin/ChatHistory 实体 |
| B | 建 `song`/`playlist` 表，写实体类，插 20 首种子数据 | 表建好，有数据 |
| C | 建 `user_behavior`/`daily_recommend` 表，写实体类 | 表建好 |
| D | Gateway 路由规则配好 | `/chat/**` `/music/**` `/rec/**` `/user/**` 全部路由 |

> ⚠️ 关键日：D 必须在今天结束前让所有人的服务注册到 Nacos。

### Day 2（周二）

| 队员 | 任务 | 产出 |
|:---|:-----|:-----|
| D | JWT 工具类 + 登录接口 | 调 POST `/user/login` 能拿到 Token |
| A | ChatController 基础 CRUD | `/chat/hello` + `/chat/send` 接口能调通 |
| B | MusicController 基础 CRUD | `/music/song/list` 能返回歌曲列表 |
| C | RecController 基础 CRUD | `/rec/hello` + `/rec/hot` 接口能调通 |
| D | 注册接口 + Redis 验证码（模拟） | `/user/register` 能注册用户 |

### Day 3（周三）

| 队员 | 任务 |
|:---|:-----|
| D | JWT 拦截器，保护需要登录的接口 |
| A | WebSocket 配置类，前后端握手 |
| B | 歌单 CRUD（创建/编辑/删除歌单） |
| C | Redis 热门榜单逻辑（ZSET 初始化） |

### Day 4（周四）

| 队员 | 任务 |
|:---|:-----|
| D | 个人信息接口（查/改） |
| A | 对话规则引擎（关键词匹配 + 预设回复） |
| B | 播放器前端组件：封面 + 歌名 + 播放/暂停 |
| C | 热门榜单前端组件：TOP 10 列表 |

### Day 5-7（周五~周日）

|:---|:-----|
| 全员 | 各自完善代码，更新开发日志 |
| D | 测试 Gateway 全网路由 |
| — | **Week 1 检查点：4 个服务都能独立启动 + 自己的接口能调通** |

## 关键节点汇总

| 时间 | 节点 | 标准 |
|:---|:-----|:-----|
| Week 1 结束 | **各自服务能启动** | `mvn spring-boot:run` 不报错，`/hello` 返回 200 |
| Week 2 结束 | **各自模块能单独演示** | 前端面板有内容，后端接口能 CRUD |
| Week 3 结束 | **全链路打通** | 注册→对话→搜歌→播放→收藏，一个流程不断 |
| Week 4 结束 | **答辩就绪** | PPT 完整，演示稳定，日志补全 |
