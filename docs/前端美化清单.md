# 前端美化清单

本文档从“最后一周待做清单”更新为当前 UI 状态和后续打磨项。当前前端已经不是纯模板，重点应放在稳定演示、信息层级和跨模块体验。

## 1. 已完成的视觉与交互

| 项目 | 状态 | 说明 |
|:--|:--|:--|
| 单屏控制台布局 | 已完成 | 顶栏、三栏主区、底栏 |
| 聊天气泡样式 | 已完成阶段版 | 区分用户和 DJ，支持推荐歌曲卡片 |
| 推荐歌曲点击播放 | 已完成 | `dj-play-song` 事件联动 MusicPanel |
| 天气组件 | 已完成阶段版 | 实时/演示来源、定位、悬浮详情 |
| 时间组件 | 已完成阶段版 | 顶栏时间日期、悬浮完整信息 |
| 音乐面板 | 已完成阶段版 | 播放器、列表、歌单、歌词、情绪信息 |
| 推荐面板 | 已完成阶段版 | 热榜、每日推荐、偏好标签 |
| 基础响应式 | 已有基础 | 仍需实际多尺寸截图检查 |

## 2. 答辩前优先打磨

| 优先级 | 项目 | 目标 |
|:--|:--|:--|
| P0 | 空状态和错误状态 | 接口失败、无歌、无推荐、无播放 URL 都要清楚 |
| P0 | 启动后首屏稳定 | 后端某个模块未启动时前端不白屏 |
| P0 | 文案统一 | 不要出现“假成功”或和真实状态冲突的文案 |
| P1 | 推荐卡片交互 | 鼠标悬浮、点击反馈、播放中状态更清楚 |
| P1 | 播放器状态 | 无版权/无 URL、加载中、播放失败要有提示 |
| P1 | 天气和时间悬浮层 | 内容不要遮挡主操作，移动到视口内 |
| P2 | 小屏布局 | 窄屏下三栏折叠为纵向，按钮和文字不溢出 |

## 3. 不建议现在做

- 大规模重写 `App.vue` 布局；
- 新增另一套 UI 框架；
- 把项目改成多页面后台；
- 为了视觉效果引入大量装饰但不解决功能体验；
- 在用户模块未完成前，把登录 UI 包装成真实鉴权。

## 4. 演示检查清单

答辩前建议按这个顺序截图或录屏：

1. 首页完整控制台；
2. 天气悬浮详情；
3. 时间悬浮详情；
4. 普通聊天；
5. 带天气/时间/音乐工具的 AI 对话；
6. 点击聊天推荐歌曲后播放器变化；
7. 网易云搜索和播放 URL 获取；
8. 歌单导入；
9. 歌词和情绪信息；
10. 热门榜、今日推荐、偏好标签。
