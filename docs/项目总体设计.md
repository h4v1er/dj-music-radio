## 二、最终产品长什么样

## 三、这个项目到底解决了什么问题

| 普通人听歌 | 我们的 DJ 电台 |
|:---|:-----|
| 打开音乐 App，搜歌、手动找 | 打开电台，DJ **主动和你对话**，根据天气、时间、偏好推荐 |
| 推荐算法看不懂 | 推荐理由直接说出来："早上好，今天北京下雨☔，来点舒缓的" |
| 歌单一成不变 | 每天自动更新推荐，根据你的播放行为学习 |

**一句话目标**：让用户打开网页，DJ 自动打招呼，音乐开始播放，一切都是为你定制的。

## 五、每个队员到底要交付什么

### 👤 队员A — 智能对话 DJ

| 交付物 | 验收标准 |
|:---|:-----|
| **ChatPanel.vue** | 用户输入文字 → DJ 回复 → 对话列表滚动 |
| **WeatherWidget.vue** | 显示城市 + 温度 + 天气图标（调用和风天气 API） |
| **WebSocket 聊天** | 前端 ↔ 后端实时通信，刷新页面保留最近 10 条 |
| **DJ 欢迎语** | 早/中/晚不同问候："早上来点活力音乐？" |
| **module-chat 后端** | 至少 3 个接口：hello + send + history + weather |

> **最终效果**：你在聊天框说"来点摇滚"，DJ 回复一段话 + 推荐 3 首歌。

---

### 👤 队员B — 音乐中心

| 交付物 | 验收标准 |
|:---|:-----|
| **MusicPanel.vue** | 完整的音乐播放器：封面 → 歌名/歌手 → 进度条 → 播放/暂停/上下曲 → 音量 |
| **歌单列表** | 显示用户歌单，点击切换 |
| **歌单导入** | 上传文件 → RabbitMQ 异步处理 → 入库 → 前端刷新 |
| **音乐搜索** | 输入关键词 → 后端查 MySQL → 返回结果列表 |
| **module-music 后端** | 至少 5 个接口：CRUD 歌曲/歌单 + 搜索 + 异步导入 |

> **最终效果**：中间面板是一个能放歌的播放器（有 20 首本地歌曲数据），能建歌单、导入歌单。

---

### 👤 队员C — 每日推荐

| 交付物 | 验收标准 |
|:---|:-----|
| **RecPanel.vue** | 热门榜单 TOP 10（从 Redis 读取）+ 今日推荐列表 + 偏好标签 |
| **热门榜单** | Redis ZSET 排序，每次播放 +1 分，定时刷新 |
| **个性化推荐** | 根据用户收藏/播放记录，简单规则推荐（同流派/同歌手） |
| **定时任务** | `@Scheduled` 每天凌晨生成推荐结果 |
| **用户行为采集** | 接收其他服务上报的 play/like/skip 事件 |
| **module-rec 后端** | 至少 4 个接口：hot + daily + similar + behavior |

> **最终效果**：右侧面板显示实时热门榜单 + 每天更新的个性化推荐 + 偏好标签。

---

### 👤 队员D — 用户中心 + 基础设施

| 交付物 | 验收标准 |
|:---|:-----|
| **UserBar.vue** | 底部状态栏 + 登录弹窗 + 收藏按钮 + 历史按钮 |
| **JWT 登录注册** | 注册 → 登录 → 返回 Token → 前端存储 → 每次请求携带 |
| **收藏管理** | 收藏/取消收藏歌曲，列表展示 |
| **播放历史** | 记录最近播放的 50 首歌 |
| **Redis 验证码** | 注册时发验证码（模拟），存 Redis，5 分钟过期 |
| **Nacos 配置中心** | 至少一个共享配置（如天气 API KEY）放 Nacos |
| **Gateway 网关** | 路由规则 + 跨域配置 + Nacos 服务发现 |
| **module-user 后端** | 至少 6 个接口：register + login + info + favorites + history |

> **最终效果**：登录后底部显示用户名，能收藏歌曲，能在导航栏看到自己的播放历史。Gateway + Nacos 全部跑通。

## 六、完成度分级

答辩时根据实际情况选：

| 级别 | 描述 | 技术栈覆盖 |
|:---|:-----|:---|
| 🥉 **基础**（必达） | 4 个服务都能启动，前端三栏布局，每栏有内容 | Spring Boot + MyBatis Plus + VUE3 |
| 🥈 **良好**（争取） | 基础 + DJ 对话有回复 + 播放器能放歌 + 榜单有数据 | + WebSocket + Redis + RabbitMQ |
| 🥇 **优秀**（加分） | 良好 + 推荐算法有逻辑 + 歌单异步导入 + Nacos 配置 | + Spring Cloud 全套 |

## 八、音乐来源（关键问题解决）

```
┌─────────────────────────────────────────────┐
│  数据库预置 20 首歌曲（纯数据，无版权问题）    │
│                                             │
│  id  title          artist    genre          │
│  1   示例歌曲1      歌手A     摇滚            │
│  2   示例歌曲2      歌手B     电子            │
│  ...                                        │
│                                             │
│  演示方案：                                  │
│  · 本地准备 3-5 个 mp3 示例音频（免费素材）    │
```
