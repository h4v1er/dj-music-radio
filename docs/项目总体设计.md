# DJ 音乐电台项目总体设计

本文档说明项目“做成什么样、各模块边界是什么、当前真实完成度如何”。更详细的答辩材料见 [项目完整说明书.md](./项目完整说明书.md)，新人运行步骤见 [新人运行手册.md](./新人运行手册.md)。

## 1. 产品定位

DJ 音乐电台是一个单屏网页音乐控制台，不是传统后台管理系统。用户打开一个页面即可完成 AI 对话、天气时间查看、歌曲搜索播放、歌单导入、歌词情绪分析、热门榜和每日推荐。

当前前端真实入口是 `frontend/src/App.vue`：

```text
顶部：标题、当前时间、天气
左侧：AI DJ 对话
中间：播放器、歌曲列表、歌单、歌词、情绪信息
右侧：热门榜、今日推荐、偏好标签
底部：用户状态栏
```

## 2. 当前集成状态

| 模块 | 当前状态 | 说明 |
|:--|:--|:--|
| `gateway` | 基础路由已完成 | 统一转发 `/chat/**`、`/music/**`、`/rec/**`、`/user/**` |

## 4. 核心业务流程

### 4.1 AI DJ 对话

```text
用户输入
  -> ChatPanel 通过 WebSocket 或 REST 发送
  -> ChatService 读取历史和前端上下文
  -> DeepSeek 规划可用工具
  -> 后端执行 music / rec / weather / time 工具
  -> 如需定位，WebSocket 请求前端执行 location.current
  -> DeepSeek 基于真实工具结果生成回复
  -> 返回 reply、selectedSongs、toolCalls
  -> 用户点击推荐歌曲，MusicPanel 播放
```

设计原则：AI 可以决定调用哪些工具，但歌曲、天气、时间等事实数据必须来自系统工具，不让 AI 随意编造。

### 4.2 音乐播放和歌单导入

```text
网易云搜索或本地歌库
  -> MusicPanel 展示歌曲
  -> NeteaseCloudMusicApi 获取播放 URL / 歌词 / 歌单
  -> 播放行为写入历史或上报推荐

歌单导入
  -> 前端解析网易云歌单或 JSON
  -> POST /music/playlist/import
  -> RabbitMQ
  -> PlaylistImportListener 写入 song / playlist / playlist_song
  -> 获取歌词并触发情绪分析
```

### 4.3 推荐链路

```text
用户行为 play / like / share / skip
  -> POST /rec/behavior
  -> 写 MySQL user_behavior
  -> 更新 Redis ZSET song:hot:rank

每天 02:00
  -> RecommendScheduler
  -> 基于用户 Top 行为歌曲生成 daily_recommend
  -> RabbitMQ 发送推荐更新通知
```

## 5. 分工边界

| 队员 | 分支 | 模块 | 当前文档口径 |
|:--|:--|:--|:--|
| A | `dev-chat` | 智能对话 DJ、天气、时间、定位、聊天联动 | 当前分支同时承担集成 music/rec 的职责 |
| B | `dev-music` | 音乐中心、网易云、歌单、歌词、情绪、收藏历史 | 已合入当前集成分支 |
| C | `dev-rec` | 热榜、每日推荐、相似推荐、偏好标签 | 已合入当前集成分支 |
| D | `dev-user` | 用户中心、JWT 会话、用户资料 | 已合入当前集成分支；Gateway 全局鉴权过滤器仍属于后续增强 |

## 7. 当前主要风险

| 风险 | 处理方式 |
|:--|:--|
| Gateway 还没有统一鉴权过滤器 | 当前由 `module-user` 在自身接口校验 JWT；其他模块仍主要通过 `userId` 参数联动 |
| Java 版本不一致 | 父 POM 为 Java 25，远端默认 Java 17，构建使用 JDK25 脚本 |
| 外部 key 依赖 | DeepSeek、和风天气全部使用环境变量，不提交真实 key |
| 网易云版权限制 | 部分歌曲无播放 URL 是外部版权限制，前端需要允许失败提示 |
| 本地数据库不随 Git 同步 | 新人需执行 SQL，并通过网易云或 JSON 导入歌库数据 |

## 8. 答辩表达重点

- 项目采用 Spring Cloud 多模块微服务架构；
- AI 对话不是纯模板，而是“AI 规划工具 + 系统执行工具 + AI 总结回复”；
- 音乐模块连接真实网易云代理，同时保留本地歌库、歌词、情绪、品味能力；
- 推荐模块有 Redis 热榜、行为采集、相似推荐、每日定时任务；
- 用户模块已接真实注册登录和 JWT，会话变化会刷新聊天、音乐和推荐模块；
- 前端是单屏 DJ 控制台，聊天推荐歌曲可以直接点击播放。
