## 2. 项目背景

当前常见音乐产品通常把“搜索、播放、推荐、歌单、用户中心”拆在多个页面，用户需要自己在搜索框、推荐页、天气/场景和播放列表之间来回切换。另一方面，普通聊天 AI 如果直接推荐歌曲，容易出现编造歌名、无法播放、无法结合系统真实数据的问题。

本项目希望把 AI 对话、真实音乐数据、推荐算法和用户行为统一到一个单屏 DJ 控制台中。用户可以像和一个音乐助理聊天一样提出需求，例如“我这里下雨，推荐几首适合晚上听的歌”“今天适合听什么”“给我找周杰伦”“我想听治愈一点的”，系统再根据可用工具调用音乐、推荐、天气、时间、定位等真实服务，返回可点击播放的歌曲。

项目的核心目标不是做一个静态播放器，而是做一个能联动数据的智能音乐助手：

- AI 负责理解用户意图、规划工具、组织自然回复；
- 歌曲数据来自本地歌库、网易云代理或推荐模块，不让 AI 随意编造；
- 推荐结果来自用户播放、收藏等行为；
- 用户登录后，聊天、音乐、推荐、收藏、历史和播放队列都能按当前用户联动；
- 答辩机上可以通过一键脚本启动完整演示环境。

## 3. 团队分工

### 3.1 队员 A：智能对话 DJ 与项目整合

负责内容：

- `module-chat` 后端服务，端口 8081；
- `ChatService`：对话主流程、历史记录、工具执行、候选歌曲选择；
- `DeepSeekChatClient`：DeepSeek 工具规划和最终回复；
- `WeatherService`：和风天气实时 API，失败时明确返回演示数据；
- WebSocket `/chat/ws`：实时对话和前端工具请求；
- `ChatPanel.vue`：聊天气泡、推荐歌曲卡片、WebSocket/REST 降级、浏览器定位工具；
- `WeatherWidget.vue` 和 `TimeWidget.vue`：顶栏天气、时间和悬浮详情；
- 合并并整合 `dev-music`、`dev-rec`、`dev-user` 到当前 `dev-chat`；
- 修复跨模块联动问题，包括网易云代理、推荐补全真实歌名、歌单创建、播放队列持久化、用户会话刷新等；
- 编写和维护运行文档、答辩文档、一键启停脚本说明。

### 3.2 队员 B：音乐中心

负责内容：

- `module-music` 后端服务，端口 8082；
- 歌曲接口：分页、搜索、详情、流派列表、歌词保存；
- 网易云代理：搜索、播放 URL、详情、歌词、歌单、封面代理、健康检查；
- 歌单：创建、更新、删除、歌曲添加/移除、排序、异步导入；
- RabbitMQ 歌单导入链路；
- 歌词情绪分析：DeepSeek 优先，关键词规则兜底；
- 用户收藏、播放历史、用户品味画像；
- 播放队列持久化：`user_play_queue`、`user_player_state`；
- 前端 `MusicPanel.vue` 及播放器、列表、歌单、歌词、情绪相关组件。

### 3.3 队员 C：每日推荐

负责内容：

- `module-rec` 后端服务，端口 8083；
- 用户行为上报 `/rec/behavior`；
- Redis ZSET 热门榜 `/rec/hot`；
- 今日推荐 `/rec/daily`；
- 手动刷新 `/rec/daily/refresh`；
- 相似推荐 `/rec/similar`；
- 偏好标签 `/rec/preferences`；
- OpenFeign 调用音乐模块补全歌曲信息；
- 定时任务每天 02:00 生成每日推荐；
- RabbitMQ 发送推荐更新通知；
- 前端 `RecPanel.vue` 展示热榜、今日推荐、偏好标签和刷新按钮。

### 3.4 队员 D：用户中心

负责内容：

- `module-user` 后端服务，端口 8084；
- 用户注册、登录；
- BCrypt 密码哈希；
- JWT 签发和校验；
- 用户资料查询和更新；
- 修改密码；
- 用户入口的收藏列表、取消收藏、播放历史；
- 通过 OpenFeign 调用 `module-music`，收藏和历史数据不重复建表；
- 前端 `UserBar.vue` 登录/注册/退出/修改密码、收藏/历史弹窗。

## 4. 竞品分析

| 竞品/类型 | 优点 | 不足 | 本项目对应设计 |
|:--|:--|:--|:--|
| 网易云音乐、QQ 音乐等成熟播放器 | 歌曲版权、播放体验、歌单生态成熟 | AI 对话和课程项目级可控后端联动不是重点；推荐算法和数据链路对学生不可见 | 通过 NeteaseCloudMusicApi 接入网易云搜索、歌词和歌单，同时保留自建后端表、行为记录和推荐逻辑 |
| 普通 AI 聊天工具 | 自然语言理解能力强，回答灵活 | 直接推荐歌曲可能编造，无法保证能播放，也无法读取项目数据库 | DeepSeek 只负责规划和回复，歌曲、天气、推荐必须来自工具结果 |
| 简单 Vue 音乐播放器项目 | 前端展示直观，容易演示 | 通常缺少用户体系、推荐算法、微服务架构和真实外部服务 | 本项目拆成 chat/music/rec/user/gateway，配合 MySQL、Redis、RabbitMQ、Nacos |
| 传统后台管理系统 | 模块边界清楚，CRUD 完整 | 音乐场景体验弱，用户需要多页面切换 | 单屏控制台：聊天、播放器、推荐、天气、用户状态同时可见 |

本项目优势：

- 使用微服务拆分，适合四人分工；
- AI 对话与真实工具调用结合，避免纯模板；
- 音乐模块有本地数据和网易云代理两类来源；
- 推荐模块能从用户行为、Redis 热榜和 MySQL 推荐表形成闭环；
- 用户模块已经接入 JWT，会话变化能刷新聊天、音乐、推荐；
- 一键启动脚本降低答辩演示风险。

当前不足：

- Gateway 尚未实现全局鉴权过滤器；
- 用户模块没有验证码、找回密码和真实头像文件上传；
- 歌曲版权受网易云外部接口限制，部分歌曲可能无法返回播放 URL；
- 数据库内容不随 Git 同步，新环境需要执行 SQL 并导入歌单数据；
- 推荐效果依赖足够的播放/收藏行为和歌曲情绪标签。

## 5. 界面原型与当前界面

当前前端真实入口是 `frontend/src/App.vue`，不是旧路由页。页面采用单屏 DJ 控制台布局：

```text
顶部栏：项目标题、当前时间、天气
左侧：AI DJ 对话
中间：播放器、歌曲列表、网易云搜索、歌单、歌词、情绪信息
右侧：热门榜、今日推荐、偏好标签
底部：用户状态栏、登录注册、收藏历史、修改密码
```

关键交互：

- Chat 面板支持 WebSocket 实时对话，失败时 REST 降级；
- AI 推荐歌曲显示为可点击卡片，点击后触发 `dj-play-song`，由 MusicPanel 播放；
- WeatherWidget 优先浏览器定位，失败时使用默认城市北京，悬浮显示体感、湿度、风、气压、能见度等；
- TimeWidget 顶栏显示日期时间，悬浮显示完整时间和时区；
- MusicPanel 支持本地歌曲、网易云搜索、歌单、收藏、历史、歌词、情绪、播放队列；
- 播放队列可展开，刷新页面后按用户恢复；
- PlayerCore 点击封面可打开完整播放详情；
- RecPanel 支持点击推荐歌曲播放，并支持手动刷新今日推荐；
- UserBar 支持登录、注册、退出、修改密码、收藏/历史弹窗。

## 8. 数据库设计

### 8.1 `mall` 数据库

| 表 | 模块 | 说明 |
|:--|:--|:--|
| `chat_history` | chat | 对话历史，保存用户与 assistant 消息，按 `user_id/create_time` 查询最近 10 条 |
| `dj_user` | user | 用户身份资料，包含 username、password_hash、nickname、avatar、phone、email |
| `user_behavior` | rec | 用户播放/收藏/分享/跳过等行为，用于推荐和热榜 |
| `daily_recommend` | rec | 每日推荐结果，按 user_id + push_date 查询 |

### 8.2 `dj_music_radio` 数据库

| 表 | 说明 |
|:--|:--|
| `song` | 歌曲表，包含标题、歌手、专辑、流派、封面、来源、来源 ID、歌词、播放次数、情绪冗余字段 |
| `playlist` | 用户歌单 |
| `playlist_song` | 歌单歌曲关联和排序 |
| `user_favorite` | 用户收藏歌曲 |
| `play_history` | 用户播放历史 |
| `user_play_queue` | 用户播放队列快照 |
| `user_player_state` | 用户当前播放模式和当前歌曲标识 |
| `song_emotion` | 歌曲情绪画像 |
| `emotion_keyword` | 情绪关键词词典 |
| `user_taste` | 用户音乐品味画像 |

### 8.3 数据边界

- 用户身份只保存在 `mall.dj_user`；
- 收藏、历史、播放队列统一由 `module-music` 维护；
- `module-user` 只在校验 JWT 后调用 music，不重复建立收藏/历史表；
- `module-rec` 根据 userId 和 songId 记录行为，不保存用户密码或资料；
- 根目录旧 `src/main/java/org/example/mall` 是历史模板代码，不属于当前父 POM 多模块主线。

## 10. 核心业务流程

### 10.1 AI 对话推荐

```text
用户输入
  -> ChatPanel 通过 WebSocket 或 REST 发送
  -> ChatService 读取历史和 context.location
  -> DeepSeek 规划工具
  -> 后端执行 music/rec/weather/time 工具
  -> 如需浏览器定位，WebSocket 返回 tool_request
  -> DeepSeek 基于工具结果生成自然回复
  -> 返回 selectedSongs
  -> 用户点击歌曲
  -> MusicPanel 播放并记录历史、上报推荐行为
```

### 10.2 音乐播放与队列恢复

```text
用户播放歌曲
  -> MusicPanel 设置 currentSong
  -> 加入 playQueue
  -> PUT /music/queue/state
  -> user_play_queue / user_player_state
  -> 页面刷新
  -> GET /music/queue/state
  -> 恢复队列和播放模式
```

### 10.3 推荐闭环

```text
播放/收藏
  -> POST /rec/behavior
  -> user_behavior
  -> Redis song:hot:rank
  -> /rec/hot 展示热榜

点击刷新或定时任务
  -> Top 行为歌曲
  -> 情绪标签/流派/歌手/协同过滤/专辑/时长
  -> daily_recommend
  -> /rec/daily 展示今日推荐
```
