# 组员开发手册

本文档是团队日常协作规范。运行环境的完整步骤见 [新人运行手册.md](./新人运行手册.md)，接口细节见 [API接口规范.md](./API接口规范.md)。

## 1. 仓库和分支

仓库：

```text
git@github.com:h4v1er/dj-music-radio.git
```

当前集成分支：

```text
dev-chat
```

个人开发分支约定：

| 队员 | 分支 | 负责范围 |
|:--|:--|:--|
| A | `dev-chat` | 智能对话 DJ、天气、时间、定位、跨模块集成 |
| B | `dev-music` | 音乐中心、网易云、歌单、歌词、情绪、收藏历史 |
| C | `dev-rec` | 推荐、热榜、行为、每日推荐、偏好标签 |
| D | `dev-user` | 用户中心、登录注册、JWT、用户资料、最终鉴权 |

开工前先确认自己在哪个分支：

```bash
git branch --show-current
git status --short
```

## 2. 远端 Windows 开发模式

正式开发运行环境在远端 Windows：

```text
D:\projects\dj-music-radio
```

Mac 上的 sshfs 挂载目录：

```text
/Users/xuwangweifan/Desktop/项目实训/最终项目/dj-music-radio-sshfs
```

使用原则：

| 操作 | 应该在哪里做 |
|:--|:--|
| 读代码、改代码、改文档 | sshfs 挂载目录可以做 |
| `git status` / `git diff` / `git commit` / `git push` | 远端 Windows 仓库 |
| Maven 构建、npm 构建、启动服务、看日志 | 远端 Windows |
| MySQL、Redis、RabbitMQ、Nacos 检查 | 远端 Windows |

原因：sshfs 适合文件读写，但直接在 Mac 挂载目录跑 Git 可能受文件元数据、换行、大小写和性能影响，容易误判大量修改。

## 3. 环境和版本

当前父 POM：

```xml
<java.version>25</java.version>
```

远端默认 `java` 可能仍是 JDK17，因此后端构建和启动优先用远端 JDK25 脚本：

```powershell
cd D:\projects\dj-music-radio
powershell -ExecutionPolicy Bypass -File D:\projects\mvn25-dj-music-radio.ps1 clean package -DskipTests
```

不要为了本机方便单独改 `pom.xml` 的 Java 版本后提交。需要改版本时必须全队统一。

## 4. 必备环境变量

不能把真实密码和 API key 写进 Git。当前配置通过环境变量读取：

```powershell
[Environment]::SetEnvironmentVariable("MYSQL_PASSWORD", "你的MySQL密码", "User")
[Environment]::SetEnvironmentVariable("DEEPSEEK_API_KEY", "你的DeepSeek Key", "User")
[Environment]::SetEnvironmentVariable("QWEATHER_API_KEY", "你的和风天气Key", "User")
[Environment]::SetEnvironmentVariable("QWEATHER_API_HOST", "你的和风天气专属Host", "User")
[Environment]::SetEnvironmentVariable("JWT_SECRET", "至少32字节的用户JWT密钥", "User")
```

RabbitMQ 如果不是默认 `guest/guest`：

```powershell
[Environment]::SetEnvironmentVariable("RABBITMQ_USERNAME", "guest", "User")
[Environment]::SetEnvironmentVariable("RABBITMQ_PASSWORD", "guest", "User")
```

改完环境变量后重启终端和对应服务。

## 5. 日常开发流程

```bash
# 1. 开工前同步目标分支
git fetch origin
git checkout dev-chat
git pull origin dev-chat

# 2. 开发和自测
# 后端至少跑对应模块 package；前端改动至少 npm run build

# 3. 提交前检查
git status --short
git diff --stat

# 4. 提交
git add <本次相关文件>
git commit -m "feat(chat): 描述本次改动"
git push origin dev-chat
```

如果是在个人分支开发：

```bash
git checkout dev-music
git pull origin dev-music
git merge origin/dev-chat
```

冲突必须按真实代码解决，不能直接覆盖别人的模块。

## 6. 提交消息规范

使用：

```text
feat(chat): 新增天气工具
fix(music): 修复网易云播放 URL 为空提示
docs: 更新新人运行手册
style(frontend): 调整聊天推荐卡片样式
refactor(rec): 整理推荐服务查询逻辑
```

避免：

```text
update
修改
111
.
```

## 7. 模块联调边界

| 模块 | 联调时必须确认 |
|:--|:--|
| chat | `/chat/send`、`/chat/ws`、`/chat/weather`；DeepSeek 和天气 key 是否配置 |
| music | `NeteaseCloudMusicApi:3000` 是否启动；`dj_music_radio` 是否有数据 |
| rec | Redis 是否可用；`user_behavior` / `daily_recommend` 表是否存在 |
| user | `/user/register`、`/user/login`、`/user/info`；JWT 是否有效；收藏/历史是否能通过 music 返回 |
| frontend | Vite 代理 `/api` 到 8080，`/music` 到 8082 |

## 8. 前端协作规范

当前真实页面是 `frontend/src/App.vue` 组合单屏控制台。可以改 `App.vue`，但必须保持单屏控制台产品形态，不要把它改回多页面入口。

组件边界：

| 文件 | 主要负责 |
|:--|:--|
| `ChatPanel.vue` | 对话、WebSocket、工具请求、聊天推荐歌曲点击播放 |
| `WeatherWidget.vue` | 顶栏天气、定位、天气详情悬浮 |
| `TimeWidget.vue` | 顶栏时间日期、悬浮详情 |
| `MusicPanel.vue` | 播放器、歌曲列表、歌单、歌词、情绪信息 |
| `RecPanel.vue` | 热榜、今日推荐、偏好标签 |
| `UserBar.vue` | 底部用户状态、登录注册、退出、收藏/历史弹窗 |

样式优先复用 `frontend/src/styles/variables.css` 和 `global.css` 的变量、按钮、面板样式。

## 9. 自测要求

按改动范围选择：

| 改动 | 最低自测 |
|:--|:--|
| Java 后端 | `mvn25-dj-music-radio.ps1 -q -DskipTests package` 或对应模块 `-pl xxx -am package` |
| 前端 | `npm.cmd run build` |
| API 改动 | 用 Gateway 路径实际请求一次 |
| chat 工具 | 验证普通聊天、音乐推荐、天气、定位至少一种 |
| music | 验证 `/music/netease/ping`、搜索、播放 URL 或歌单导入 |
| rec | 验证 `/rec/hot`、`/rec/behavior`、`/rec/preferences` |
| user | 验证注册、登录、带 token 访问 `/user/info`、收藏/历史列表 |
| 文档 | 检查是否和真实代码、端口、环境变量一致 |

## 10. 常见问题

| 现象 | 原因 | 处理 |
|:--|:--|:--|
| Maven 报 Java 版本不支持 | 默认 Java 是 17，但 POM 是 25 | 用 JDK25 脚本 |
| 前端能打开但接口失败 | Gateway 或后端模块没启动 | 先测 `/chat/hello`、`/music/hello` |
| 网易云搜索失败 | 3000 代理没启动 | 启动 `scripts/netease/start-netease-api.ps1` |
| 天气是演示数据 | 和风 key/host 未配置或失败 | 配环境变量后重启 chat |
| AI 变成本地兜底 | DeepSeek key 未配置或调用失败 | 检查环境变量和日志 |
| 推荐为空 | Redis 或行为数据为空 | 先上报播放/收藏行为 |
| 登录 401 | 密码错误、token 缺失或 token 过期 | 重新登录，确认请求头有 `Authorization: Bearer <token>` |
| 用户收藏/历史接口失败 | module-user 会调用 module-music | 先确认 module-music 已启动并注册到 Nacos |
