# 📋 组员开发手册（每人必读）

> 仓库：https://github.com/h4v1er/dj-music-radio  
> 架构：Spring Cloud 微服务 + VUE3 桌面式控制台

---

## 第一步：克隆 & 创建自己的分支

```bash
git clone https://github.com/h4v1er/dj-music-radio.git
cd dj-music-radio

# ⚠️ 先设好自己的身份！截图才能证明是你提交的
git config user.name "你的姓名拼音"
git config user.email "你的邮箱"

# 创建你的开发分支（选自己的）
git checkout -b dev-chat     # 队员A
git checkout -b dev-music    # 队员B
git checkout -b dev-rec      # 队员C
git checkout -b dev-user     # 队员D
```

---

## 第二步：启动环境

```bash
# 中间件（已装好）
brew services start mysql redis rabbitmq

# Nacos
bash ~/environment/nacos/bin/startup.sh -m standalone

# 后端：启动你的微服务
mvn spring-boot:run -pl module-xxx    # 改成你的模块名

# 前端
cd frontend && npm install && npm run dev

```

如果需要联调音乐中心的网易云搜索、播放、歌词或歌单导入，还必须额外启动本机 `NeteaseCloudMusicApi:3000`：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/netease/start-netease-api.ps1
```

详细说明见 [module-music-runtime.md](./module-music-runtime.md)。

## 第四步：日常开发流程

```bash
# 每次开工前
git pull origin master          # 拉最新代码

# 写代码...

# 收工前提交
git add .
git commit -m "feat(模块): 做了什么"   # 格式见下方
git push origin dev-xxx               # 推到你的分支

# 更新开发日志
# 打开 docs/devlogs/队员X-xxx.md，写下今天干了什么
git add docs/devlogs/
git commit -m "docs: 更新开发日志"
git push origin dev-xxx
```

---

## 第五步：提交消息格式

```
feat(chat): 实现WebSocket实时对话      ← 新功能
fix(music): 修复播放器进度条bug         ← 修bug
docs: 更新开发日志                      ← 文档
style: 调整面板间距                     ← 样式
```

**禁止**写：`update`、`修改`、`1111`、`.` 这种无意义的提交。

## 第六步：前端规范速查

```css
/* 颜色 — 只能用变量，禁止硬编码 #xxx */
var(--color-bg)         /* 页面背景 */
var(--color-surface)    /* 面板背景 */
var(--color-primary)    /* 红色主色调 */
var(--color-text)       /* 正文 */
var(--color-text-muted) /* 次要文字 */

/* 面板 — 统一用这个类 */
<div class="panel">
  <div class="panel-title"><span class="dot"></span>标题</div>
  <!-- 你的内容 -->
</div>

/* 按钮 */
<button class="btn-primary">主要操作</button>
<el-button type="primary">Element Plus按钮</el-button>
```

⚠️ **禁止改的文件**：`App.vue`（布局）、`variables.css`（颜色变量）、`global.css`（全局样式）

## 第七步：接口联调

各模块接口约定见 `docs/API接口规范.md`。  
前端调后端通过 Vite 代理，写法：

```js
import { chatApi } from './api'
const res = await chatApi.hello()   // 自动走 http://localhost:8080/chat/hello
```

在 GitHub 网页上：

3. 写清楚你做了什么
4. 队长审核后合并

## ⚠️ 常见错误

| 错误 | 原因 | 解决 |
|:-----|:-----|:-----|
| 启动报 `common:jar` 找不到 | 没装 common 模块 | `mvn install -DskipTests` |
| 前端 5173 白屏 | Vite 没启动 | `cd frontend && npm run dev` |
| Gateway 报 Nacos 错误 | Nacos 没启动 | 先启动 Nacos |
| Git 提交记录不显示你的名字 | 没设 user.name | `git config user.name "你的拼音"` |
| 和别人改了同一个文件冲突 | `git pull` 后手动解决 | 不懂问队长 |
